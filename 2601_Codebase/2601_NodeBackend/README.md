# Fleet & Trip Management System - Backend API

This backend application orchestrates the entire fleet management process, from user authentication to complex trip generation via an external Python optimization service.

## Base URL
All endpoints are relative to the server base URL. By default:
`http://localhost:3000`

## Authentication
Most endpoints require authentication.
*   **Method**: JWT (JSON Web Token)
*   **Header**: `Authorization: Bearer <your_token_here>`

---

## 1. Authentication Module (`/auth`)
Handle user access for both Company Admins and Employees.

| Method | Endpoint | Description | Body / Notes |
| :--- | :--- | :--- | :--- |
| **POST** | `/auth/login/employee` | Login for regular staff. | `{ "email": "...", "password": "..." }` |
| **POST** | `/auth/login/company` | Login for Admin/Company accounts. | `{ "email": "...", "password": "..." }` |
| **GET** | `/auth/verify` | Validates session token. | Requires `Authorization` header. Returns user role. |
| **GET** | `/auth/google` | Starts Google OAuth implementation. | Redirects browser to Google. |
| **GET** | `/auth/google/callback` | Google OAuth callback. | Returns token after successful Google login. |
| **POST** | `/auth/logout` | Logs out the user. | Client should purge the token. |

---

## 2. Core Workflow: Excel Upload (`/upload`)
The central feature of the application. This endpoint triggers the optimization engine.

**POST** `/upload`
*   **Description**: Takes a raw Excel file containing roster data, sends it to the Python Optimizer Service, and automatically generates:
    1.  **Fleets**: A record of the uploaded schedule.
    2.  **Trips**: Optimized routes grouping multiple employees into vehicles.
*   **Headers**: `Authorization: Bearer <token>`, `Content-Type: multipart/form-data`
*   **Body (FormData)**:
    *   `file`: The Excel file (`.xlsx`).
    *   `companyId`: ID of the company optimizing the roster.
    *   `days`: JSON string array of days (e.g., `["Mon", "Tue"]`).

| **GET** | `/company/vehicles/:id` | List all owned vehicles. | Param: `id` (Company ID) |

## 5. Fleet (Roster) Management (`/fleet`)
View the high-level rosters created by the upload process.

| Method | Endpoint | Description | Filters (Query Params) |
| :--- | :--- | :--- | :--- |

## Setup & External Dependencies

1.  **Python Optimizer**:
    *   Must be running at `http://localhost:8000`.
    *   This API sends files to `http://localhost:8000/api/optimizer`.
2.  **Environment Variables**:
    *   `PORT`: Port to run this server (default 3000).
    *   `MONGO_URI`: MongoDB connection string.
    *   `JWT_SECRET`: Secret key for authentication.
